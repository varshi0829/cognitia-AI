# Cognitia AI

> A full-stack conversational AI web app — ask anything, get an instant AI-powered answer.

---

## What is Cognitia AI?

Cognitia AI is a simple, fast, and clean web application that lets users ask one question at a time and receive a response generated by an AI model. Every question and its answer is stored securely in MongoDB Atlas.

There's no chat history, no accounts, no fluff — just a question, an answer, and a beautiful interface.

## Features

- One question → one AI-generated answer

- Input validation on backend
- CORS restricted to deployed frontend URL

## Local Setup

### Prerequisites

- Node.js v18+
- A [Groq API key](https://console.groq.com)
- A [MongoDB Atlas](https://cloud.mongodb.com) cluster (free tier works)

### 1. Clone the repo

```bash
git clone https://github.com/varshi0829/cognitia-AI.git
cd cognitia-AI
```

### 2. Backend

```bash
cd backend
npm install
```

Create a `.env` file:

```env
GROQ_API_KEY=your_groq_api_key
MONGODB_URI=your_mongodb_atlas_uri
PORT=5000
FRONTEND_URL=http://localhost:5173
```

Start the server:

```bash
npm run dev
```

Runs on `http://localhost:5000`

### 3. Frontend

```bash
cd frontend
npm install
```

Create a `.env` file:

```env
VITE_API_URL=http://localhost:5000
```

Start the app:

```bash
npm run dev
```

Runs on `http://localhost:5173`

## API Reference

### `POST /api/ask`

Send a question, receive an AI-generated answer.

**Request body:**
```json
{
  "question": "What is artificial intelligence?"
}
```

**Success response** `201`:
```json
{
  "answer": "Artificial intelligence is..."
}
```

**Error response** `400`:
```json
{
  "error": "Question is required"
}
```

## Deployment

Both frontend and backend are deployed as **independent Vercel projects** from the same GitHub repository.

### Backend (deploy first)
1. New Vercel project → Root Directory: `backend`
2. Add environment variables: `GROQ_API_KEY`, `MONGODB_URI`, `FRONTEND_URL`
3. Deploy → copy the backend URL

### Frontend
1. New Vercel project → Root Directory: `frontend`
2. Add environment variable: `VITE_API_URL` = your backend URL
3. Deploy

### After both are live
Update `FRONTEND_URL` in the backend Vercel project to your frontend URL, then redeploy the backend (fixes CORS).
